## Что нужно сделать

Собрать Next.js-сайт бронирования трансферов для транспортной компании LONDON ROUTE TRANSFERS на основе клиентского текста: лендинг, страница услуг, форма бронирования с валидацией (без сохранения данных), страница списка бронирований для менеджера на моковых данных, юридические страницы. Сайт двуязычный — английский основной, русский через переключатель

## Стек

Next.js (App Router), React, TypeScript, Tailwind CSS v4, next-intl для мультиязычности, react-hook-form для формы, zod для схемы валидации. Backend не используется — все данные моковые

## Что должно получиться

- лендинг: hero, о компании, услуги (минимум 4 типа поездок), преимущества
- форма бронирования: react-hook-form + zod, состояния success/error, данные никуда не сохраняются
- страница бронирований для менеджера: список моковых записей, фильтр/поиск, empty state, не связана с формой
- юридические страницы: privacy policy, cookie policy, terms, условия оказания транспортных услуг
- server components для статичного контента (лендинг, услуги, legal), client components для формы, фильтров и переключателя языка
- SEO metadata под каждую локаль
